## 一、产品概述
### 1.1 项目背景
- 职场会议效率现状：约70%职场人对会议有不满，75%认为参加过无意义会议
- 时间浪费数据：工程师30%-50%工作时间用于会议，50%会议超时，90%参会者会走神
- 核心问题：会议信息分散、纪要整理耗时、待办事项遗漏、决策无闭环
- 市场机会：现有工具多为视频会议平台内置功能，缺乏独立轻量级、跨平台的会议助手

### 1.2 产品定位
**轻量级跨平台AI视频会议助手**
- 专注：会前准备 + 会后跟进（不做视频会议本身）
- 核心价值：让会议从"时间黑洞"变成"价值放大器"
- 差异化：跨平台兼容、极简无干扰、AI全流程赋能、待办闭环管理

### 1.3 产品目标
- 短期（1.0版本）：实现会议全流程基础功能，提升用户会议效率30%
- 中期（2.0版本）：完善AI能力，支持团队协作与知识库沉淀
- 长期：成为职场人首选的会议效率工具

## 二、用户分析
### 2.1 目标用户画像
| 用户角色   | 典型特征                       | 核心需求                                         |
| ---------- | ------------------------------ | ------------------------------------------------ |
| 会议组织者 | 产品经理、项目经理、团队leader | 快速筹备会议、确保会议高效、跟进待办落实         |
| 普通参会者 | 研发、设计、运营等执行层       | 快速了解会议内容、准确记录重点、不遗漏自己的待办 |
| 团队管理者 | 部门负责人、总监               | 掌握团队会议情况、监督决策执行、提升组织效率     |

### 2.2 全流程用户痛点分析
#### 会前痛点
- 信息分散：会议预约、资料、议程分散在不同平台（日历、微信、邮箱）
- 准备耗时：手动编写议程、收集资料、发送提醒
- 参会准备不足：参会者不知道会议主题和议程，无法提前准备
- 时间冲突：难以找到所有参会者都合适的时间
- 历史遗忘：忘记上次会议决策和待办完成情况

#### 会中痛点
- 记录困难：手动记录跟不上发言速度，容易遗漏重点
- 议程失控：缺乏明确议程节点，讨论容易跑偏，时间浪费率高达43%
- 干扰过多：视频卡顿、消息弹窗打断思路
- 发言不均：部分人主导会议，其他人难以表达观点
- 僵局浪费：同一问题讨论过久无法达成共识

#### 会后痛点
- 纪要整理耗时：2小时会议需要1-2小时整理纪要
- 信息遗漏：手动记录容易漏掉关键决策和待办事项
- 待办无闭环：待办事项无人跟进，不了了之
- 资料难查找：会议资料和纪要分散，难以检索和复用
- 跨团队记忆负担：面对多个团队的会议，难以追踪每个团队的进展

### 2.3 用户需求优先级
- P0（必须实现）：AI生成会议纪要、智能提取待办、会议资料管理
- P1（重要实现）：快速创建会议、实时语音转写、待办提醒与追踪、团队会议进展时间轴
- P2（后续实现）：日历同步、团队协作、会议数据分析

## 四、功能需求
### 4.1 会前准备模块（P0）
#### 4.1.1 快速创建会议
- 支持手动填写会议信息（主题、时间、参会人、平台、地点）
- 支持导入日历会议（Outlook、Google Calendar、企业微信日历）
- 支持一键复制会议链接和邀请信息
- 智能时间推荐：根据参会人日历推荐最佳会议时间

#### 4.1.2 AI生成会议议程
- 根据会议主题自动生成标准议程模板
- 支持自定义议程模板库
- 可设置每个议程的预计时长
- 一键发送议程给所有参会人

#### 4.1.3 会议资料管理
- 支持上传多种格式文件（PDF、Word、PPT、图片、链接）
- 资料自动分类存储，与会议关联
- 参会人可在线预览资料
- 支持资料版本管理

#### 4.1.4 会议背景智能提醒
- 自动关联历史会议：显示与本次会议相关的历史会议记录
- 上次会议决策回顾：展示上次会议达成的关键决策和待办完成情况
- 会议目标高亮：在会前突出显示本次会议的核心目标
- 时间轴预览：展示该系列会议的时间线概览

#### 4.1.5 会议提醒
- 会议前15分钟/30分钟/1小时弹窗提醒
- 支持邮件/短信/微信提醒
- 未参会人员自动提醒
- 冲突检测：自动检测时间冲突并提示用户

### 4.2 会中辅助模块（P1）
#### 4.2.1 实时语音转写
- 支持普通话/中英混合实时转写
- 自动区分发言人
- 转写内容实时保存
- 支持关键词高亮

#### 4.2.2 重点标记
- 一键标记：决策、待办、问题、重点
- 支持手动输入补充笔记
- 标记内容自动同步到纪要
- 支持截图快速插入笔记

#### 4.2.3 议程计时器与自定义提醒
- **议程时间管理**：
  - 为每个议程项设置预计时长
  - 实时显示当前议程剩余时间
  - 支持暂停/继续计时
- **自定义提醒规则**：
  - 可自定义提醒时间点（如剩余5分钟、1分钟）
  - 支持视觉提醒（颜色变化：绿色→黄色→橙色）和声音提醒
  - 可设置是否强制结束超时议程
- **AI辅助跑题检测**：
  - 通过语音转写分析讨论内容与议程主题的相关性
  - 当检测到持续跑题（超过3分钟）时，温和提醒主持人
  - 提供「拉回主题」快捷按钮
- **僵局提示**：
  - 检测到同一问题讨论超过预设时间仍未达成共识
  - 自动提示主持人「建议搁置该问题，后续单独讨论」
  - 记录问题供会后跟进
- **用户体验保障**：
  - 提醒默认仅显示给主持人，不打扰其他参会者
  - 主持人可随时开启/关闭所有提醒功能
  - 温和话术：使用"建议"而非"警告"

#### 4.2.4 小窗口悬浮
- 可调整大小和位置的悬浮窗口
- 不遮挡视频会议主画面
- 支持最小化到系统托盘

### 4.3 会后AI处理模块（P0）
#### 4.3.1 AI自动生成结构化纪要
- 自动过滤闲聊内容，提取核心信息
- 生成标准结构化纪要：
  - 会议基本信息
  - 讨论内容摘要
  - 核心结论与决策
  - 待办事项清单
- 支持自定义纪要格式和模板
- 支持编辑和修改纪要内容

#### 4.3.2 智能提取待办事项
- 自动识别待办事项、负责人、截止时间
- 待办事项可直接分配和修改
- 支持添加备注和优先级
- 待办事项与会议纪要关联，可跳转溯源

#### 4.3.3 纪要导出与分享
- 支持多格式导出：PDF、Word、Markdown、纯文本
- 一键发送纪要给所有参会人
- 支持生成分享链接，设置访问权限
- 支持导出逐字稿

### 4.4 会议管理模块（P1）
#### 4.4.1 历史会议列表
- 按时间/主题/状态筛选会议
- 会议卡片展示关键信息（时间、参会人、待办数量）
- 支持搜索会议内容
- 支持删除和归档会议

#### 4.4.2 待办事项追踪
- 待办状态管理：未开始/进行中/已完成/已逾期
- 截止时间自动提醒
- 支持批量操作待办
- 待办完成情况统计

#### 4.4.3 团队会议进展时间轴（P1）
- **按团队/项目分组**：支持按团队名称或项目名称筛选会议
- **时间轴视图**：以时间轴形式展示该团队的所有历史会议
- **进展概览**：每个会议节点显示核心决策和待办完成率
- **快速跳转**：点击时间轴节点可直接查看对应会议的完整纪要
- **趋势分析**：展示该团队会议的频次、时长、待办完成率趋势

### 4.5 数据统计模块（P2）
#### 4.5.1 会议数据分析
- 会议频次统计
- 平均会议时长分析
- 待办完成率趋势
- 超时会议占比统计

#### 4.5.2 团队协作
- 创建团队空间
- 团队成员权限管理
- 会议和待办共享
- 团队会议数据统计

## 五、非功能需求
### 5.1 性能需求
- 页面加载时间：≤2秒
- 实时转写延迟：≤1秒
- AI纪要生成速度：1小时会议≤5分钟
- 转写准确率：≥95%（普通话）

### 5.2 易用性需求
- 无需注册即可使用基础功能
- 核心操作步骤≤3步
- 提供新手引导和帮助中心
- 响应式设计，支持电脑/平板/手机

### 5.3 安全性需求
- 用户数据加密存储
- 会议内容和纪要仅用户可见
- 支持数据导出和删除
- 符合数据隐私法规要求

## 六、UI/UX设计要求
### 6.1 设计理念
- **现代AI风格**：借鉴通义听悟的设计语言，打造专业、轻盈、有科技感的产品界面
- **低干扰**：避免花哨的动画和装饰，专注于内容和功能
- **统一视觉**：建立完整的设计系统，确保所有页面风格一致
- **层次清晰**：通过颜色、字体、间距区分信息层级，提升可读性

### 6.2 参考设计：通义听悟（https://tingwu.aliyun.com/home）
通义听悟作为阿里云旗下的AI语音生产力工具，其UI设计代表了当前AI产品的主流趋势，具有极高的参考价值。以下是需要重点借鉴的设计元素：

#### 6.2.1 色彩系统
| 色彩类型        | 色值              | 使用场景                             |
| --------------- | ----------------- | ------------------------------------ |
| 主渐变          | #00B4FF → #7B61FF | 主按钮、选中状态、品牌标识、重点强调 |
| 辅助色-成功     | #00B42A           | 完成状态、成功提示                   |
| 辅助色-警告     | #FF7D00           | 提醒、待办、重点标记                 |
| 辅助色-错误     | #F53F3F           | 错误提示、删除操作                   |
| 中性色-背景     | #FFFFFF           | 页面主背景                           |
| 中性色-浅背景   | #F7F8FA           | 卡片背景、输入框背景                 |
| 中性色-边框     | #F0F2F5           | 卡片边框、分割线                     |
| 中性色-正文     | #1D2129           | 主要文字内容                         |
| 中性色-辅助文字 | #86909C           | 次要信息、提示文字                   |

**设计要点**：
- 主色调使用蓝紫渐变，体现AI产品的科技感和未来感
- 整体色彩饱和度适中，避免过于刺眼
- 色彩使用克制，仅在关键元素上使用主色和辅助色

#### 6.2.2 排版规范
- **字体**：使用无衬线字体（如Inter、PingFang SC、Microsoft YaHei）
- **字重**：标题使用500-600字重，正文使用400字重，避免使用过重的字重
- **行高**：标题行高1.2-1.3，正文行高1.5-1.6，辅助文字行高1.4
- **字体大小**：
  - 页面大标题：24px-28px
  - 模块标题：18px-20px
  - 正文：14px-16px
  - 辅助文字：12px-13px
- **间距**：
  - 页面边距：24px-32px
  - 模块间距：24px-32px
  - 元素间距：8px-16px

**设计要点**：
- 排版宽松，避免内容过于拥挤
- 建立清晰的文字层级，让用户能够快速扫描信息
- 中文文本采用左对齐，数字和英文可采用右对齐

#### 6.2.3 布局结构
- **左侧导航栏**：窄边设计（70px-80px宽），仅显示图标+文字
  - 导航项垂直排列，图标在上，文字在下
  - 选中项使用主渐变背景，文字为白色
  - 未选中项文字为深灰色，图标为灰色
- **主内容区**：
  - 采用卡片式布局，内容块之间有充足的留白
  - 卡片使用大圆角（20px），极淡的灰色边框，几乎无阴影
  - 页面顶部为页面标题和操作按钮
  - 下方为功能模块，按逻辑顺序排列
- **右侧边栏**（可选）：
  - 用于显示AI助手、资料预览、提醒等辅助信息
  - 可折叠，不占用主内容区空间

**设计要点**：
- 整体布局简洁明了，避免复杂的嵌套结构
- 大留白设计，让页面看起来更加轻盈透气
- 核心功能区域突出显示，次要功能适当弱化

#### 6.2.4 组件设计
- **按钮**：
  - 主按钮：大圆角（8px-10px），主渐变背景，白色文字
  - 次要按钮：大圆角，白色背景，灰色边框，深灰色文字
  - 文字按钮：无边框无背景，主色文字
  - 悬停效果：轻微放大（1.02倍）和亮度变化
- **卡片**：
  - 大圆角（20px）
  - 极淡的灰色边框（#F0F2F5）
  - 几乎无阴影或使用极浅的投影（rgba(0,0,0,0.03)）
  - 悬停效果：轻微上浮（2px）和阴影加深
- **输入框**：
  - 大圆角（8px）
  - 浅灰色背景（#F7F8FA）
  - 聚焦时边框变为主色
- **列表**：
  - 无分割线或使用极细的灰色分割线（#F0F2F5）
  - 列表项有悬停效果（背景变为浅灰色）
  - 选中项背景变为主色的淡色版本

**设计要点**：
- 所有组件使用统一的圆角半径和边框样式
- 组件设计简洁，避免过多的装饰元素
- 交互反馈清晰明确，让用户知道自己的操作是否成功

#### 6.2.5 交互与动效
- **页面切换**：使用平滑的淡入淡出动画
- **元素出现**：使用从下往上的滑入动画
- **按钮点击**：轻微的缩放动画
- **加载状态**：使用简洁的加载动画，避免使用复杂的GIF
- **AI生成状态**：使用渐变的脉冲动画，体现AI正在思考
- **倒计时提醒**：颜色渐变过渡（绿色→黄色→橙色），无闪烁或震动

**设计要点**：
- 动效简洁流畅，避免过于花哨的动画
- 动效时长控制在200ms-300ms之间
- 动效仅用于增强用户体验，不影响功能使用

### 6.3 页面设计规范
#### 6.3.1 首页（工作台）
- 顶部：欢迎语 + 快速创建会议按钮
- 中部：即将开始的会议卡片 + 我的待办卡片
- 底部左侧：AI会议纪要功能入口 + 历史会议快速访问

### 9.2 术语解释
- **AI转写**：利用人工智能技术将语音转换为文字的过程
- **结构化纪要**：将会议内容按照固定格式整理成清晰的文档
- **待办闭环**：从待办事项的创建、分配、执行到完成的完整流程
- **跑题检测**：通过AI分析讨论内容与议程主题的相关性
- **僵局提示**：识别长时间无法达成共识的讨论并建议搁置
